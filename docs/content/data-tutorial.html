Understanding the Synthetic NZAVS Data

A Guide to the Workshop Dataset

Published

May 1, 2025

1 Introduction

This tutorial introduces the synthetic New Zealand Attitudes and Values Study (NZAVS) dataset used in the SPARCC Day 2 workshop. The data simulates a three-wave longitudinal study examining how religious service attendance affects cooperative behaviour.

ImportantKey Point

This is synthetic data created specifically for educational purposes. While it mimics real NZAVS patterns, all observations are simulated and no real participant data is used.

2 Data Structure

2.1 Panel Design

The synthetic dataset follows a three-wave longitudinal panel:

  • Wave 10 (2018): Baseline measurements
  • Wave 11 (2019): Exposure period
  • Wave 12 (2020): Outcome measurements

2.2 Sample Size

  • Total participants: 20,000
  • Analysis sample: ~15,500 after accounting for panel attrition
  • Attrition rate: Realistic 22% dropout typical of longitudinal studies

3 Loading and Exploring the Data

3.1 Setup

Code 
# load required packages
library(tidyverse)
library(here)
library(qs)

# load the synthetic data (after running setup script)
df_nz_long <- qread(here("data", "df_nz_long.qs"))

3.2 Basic Data Exploration

Code 
# check dimensions
dim(df_nz_long)

# view first few rows
head(df_nz_long)

# check column names
colnames(df_nz_long) |> head(20)

# check time structure
table(df_nz_long$time_factor)

# participants per wave
df_nz_long |> 
  group_by(time_factor) |> 
  summarise(n_participants = n_distinct(id))

4 Key Variables

4.1 Demographics (Baseline)

Code 
# age distribution
summary(df_nz_long$age[df_nz_long$time_factor == "Time 10"])

# gender distribution
table(df_nz_long$male_binary[df_nz_long$time_factor == "Time 10"])

# ethnicity distribution
table(df_nz_long$eth_cat[df_nz_long$time_factor == "Time 10"])

# education levels
table(df_nz_long$education_level_coarsen[df_nz_long$time_factor == "Time 10"])

4.2 Exposure Variable: Religious Service

Code 
# religious service attendance distribution
baseline_religion <- df_nz_long |> 
  filter(time_factor == "Time 10") |> 
  pull(religion_church)

# view distribution
hist(baseline_religion, 
     main = "Religious Service Attendance (Baseline)",
     xlab = "Frequency (0=never, 8=weekly+)")

# binary version
table(df_nz_long$religion_church_binary[df_nz_long$time_factor == "Time 11"])

4.3 Outcome Variables: Cooperation Measures

Code 
# cooperation outcomes at Time 12
cooperation_outcomes <- c("log_charity_donate", "log_hours_charity", "support", "belong")

# summary statistics for outcomes
df_outcomes <- df_nz_long |> 
  filter(time_factor == "Time 12") |> 
  select(all_of(cooperation_outcomes))

summary(df_outcomes)

# correlation between cooperation measures
cor(df_outcomes, use = "complete.obs") |> round(2)

5 Understanding the Simulation

5.1 Realistic Features

The synthetic data incorporates realistic features of longitudinal survey data:

  1. Missing data patterns - realistic dropout and item non-response
  2. Correlated variables - demographics predict both religion and cooperation
  3. Time trends - variables show plausible change over waves
  4. Scale properties - variables maintain proper ranges and distributions

5.2 Causal Structure

The data was simulated with a realistic causal structure:

Code 
# check exposure-outcome correlation (not causal!)
religion_t11 <- df_nz_long$religion_church_binary[df_nz_long$time_factor == "Time 11"]
support_t12 <- df_nz_long$support[df_nz_long$time_factor == "Time 12"]

cor(religion_t11, support_t12, use = "complete.obs")

# but confounders matter!
age_t10 <- df_nz_long$age[df_nz_long$time_factor == "Time 10"]
cor(age_t10, religion_t11, use = "complete.obs")  # age ‚Üí religion
cor(age_t10, support_t12, use = "complete.obs")   # age ‚Üí cooperation

6 Practical Workshop Applications

6.1 Research Question

Does weekly religious service attendance increase cooperative behaviour?

6.2 Key Analysis Steps

  1. Define exposure: Religious service (binary: weekly vs. less than weekly)
  2. Define outcomes: Four cooperation measures (donations, volunteering, support, belonging)
  3. Control confounding: 32 baseline variables including demographics, personality, prior cooperation
  4. Estimate effects: Causal forests with sample splitting for heterogeneity

6.3 Expected Findings

The simulated data includes realistic effect sizes:

  • Social outcomes (support, belonging): Moderate positive effects (~0.15 SD)
  • Behavioural outcomes (donations, volunteering): Smaller positive effects (~0.10 SD)
  • Heterogeneity: Larger effects for older adults, parents, agreeable individuals

7 Data Quality Checks

7.1 Missing Data Patterns

Code 
# visualise missing data
library(naniar)

# missing data by wave
df_nz_long |> 
  group_by(time_factor) |> 
  summarise(
    pct_missing_religion = mean(is.na(religion_church)) * 100,
    pct_missing_support = mean(is.na(support)) * 100,
    pct_missing_belong = mean(is.na(belong)) * 100
  )

# pattern of missingness
vis_miss(df_nz_long |> select(id, time_factor, religion_church, support, belong))

7.2 Longitudinal Consistency

Code 
# check same individuals across waves
id_counts <- df_nz_long |> 
  group_by(id) |> 
  summarise(n_waves = n_distinct(time_factor)) |> 
  count(n_waves)

print(id_counts)

# stability of demographics over time (should be stable)
df_nz_long |> 
  group_by(id) |> 
  summarise(
    age_stable = n_distinct(age, na.rm = TRUE) <= 1,
    gender_stable = n_distinct(male_binary, na.rm = TRUE) <= 1
  ) |> 
  summarise(
    pct_age_stable = mean(age_stable) * 100,
    pct_gender_stable = mean(gender_stable) * 100
  )

8 Working with the Data

8.1 Creating Analysis Datasets

The workshop scripts demonstrate how to:

  1. Filter to eligible participants (those with baseline religion data)
  2. Create binary exposures with theory-driven cutpoints
  3. Handle missing data through multiple imputation
  4. Create wide format for causal analysis
  5. Apply IPCW weights for attrition bias

8.2 Common Data Operations

Code 
# create baseline dataset
baseline_data <- df_nz_long |> 
  filter(time_factor == "Time 10") |> 
  select(id, age, male_binary, education_level_coarsen, religion_church, support, belong)

# create change scores
change_data <- df_nz_long |> 
  filter(time_factor %in% c("Time 10", "Time 12")) |> 
  select(id, time_factor, support, belong) |> 
  pivot_wider(names_from = time_factor, values_from = c(support, belong)) |> 
  mutate(
    support_change = `support_Time 12` - `support_Time 10`,
    belong_change = `belong_Time 12` - `belong_Time 10`
  )

9 Key Takeaways

9.1 For Workshop Participants

  1. Realistic complexity - The data includes realistic confounding and missing data patterns
  2. Educational focus - Effects are large enough to detect but require proper causal methods
  3. Reproducible - Same synthetic data ensures consistent workshop results
  4. Safe to share - No privacy concerns as all data is simulated

9.2 For Analysis

  1. Three-wave minimum - Necessary for temporal ordering (confounder ‚Üí exposure ‚Üí outcome)
  2. Rich confounders - 32 baseline variables provide strong confounding control
  3. Multiple outcomes - Allows investigation of domain-specific effects
  4. Realistic heterogeneity - Some participants respond more than others

10 Technical Notes

10.1 Data Generation Process

The synthetic data was created using: - Marginal distributions matching real NZAVS data - Realistic correlation structures between variables - Appropriate temporal dependencies - Missing data mechanisms consistent with survey research

10.2 File Format

  • Format: Parquet/QS for efficient storage and loading
  • Size: ~50MB compressed
  • Structure: Long format (one row per person-wave)
  • Encoding: UTF-8 with proper factor level labels
